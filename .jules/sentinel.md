## 2025-02-14 - Secure File Creation for Secrets
**Vulnerability:** The SearXNG configuration settings file (`searxng_settings_{pid}.yml`), which contains the `secret_key` for Flask session management, was written using `Path.write_text()`. This relies on the system's default umask, potentially creating the file with world-readable permissions (e.g., `0o644`). This could expose the session secret to other users on the same machine, leading to session hijacking.
**Learning:** `Path.write_text()` or `open(..., 'w')` do not guarantee secure file permissions, making them unsafe for writing secrets or configuration files that contain sensitive credentials.
**Prevention:** Always enforce explicit and strict file permissions when creating files containing secrets. Before writing, ensure the parent directory is restricted (`chmod 0o700`). Use `os.open()` with flags `os.O_WRONLY | os.O_CREAT | os.O_TRUNC` and an explicit mode of `0o600`, followed by `os.fdopen()`, to ensure the file is created with read/write access restricted solely to the owner.
